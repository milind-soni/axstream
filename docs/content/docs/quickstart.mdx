Everything below starts from nothing. You need
[uv](https://docs.astral.sh/uv/) and [Homebrew](https://brew.sh).

## 1. Get axstream

```sh
git clone https://github.com/milind-soni/axstream
cd axstream
uv sync
```

## 2. Dry demo — no keys, no server, 10 seconds

```sh
uv run python demo_dry.py
```

Replays a canned LLM response at realistic decode speed into the real compiler
and executor against a mock computer, and prints the streamed-vs-buffered
timeline — typically ~37% wall-clock saved. This is the core thesis with zero
setup.

## 3. Set up the local pieces (once)

Three local processes power the real thing. `python -m axstream --doctor`
tells you at any point what's missing.

<Steps>

<Step>

### The tiny matcher (instant tier)

```sh
brew install llama.cpp
mkdir -p ~/models && curl -L -o ~/models/lfm25-350m-axstream-Q4_K_M.gguf \
  "https://huggingface.co/milsoni201/lfm25-350m-axstream-matcher/resolve/main/lfm25-350m-axstream-Q4_K_M.gguf"
llama-server -m ~/models/lfm25-350m-axstream-Q4_K_M.gguf --port 8791 -ngl 99 -c 4096 --no-webui
```

<Callout title="This is the fine-tuned matcher">
[`axstream-matcher`](https://huggingface.co/milsoni201/lfm25-350m-axstream-matcher)
scores 94% end-to-end on match + slot extraction (the stock base model scores
~47%). Misses are safe — they fall through to the LLM tier. Any
OpenAI-compatible server works; point `AXSTREAM_TINY_URL` at it.
</Callout>

</Step>

<Step>

### The executor: cua-driver

Background, pid-addressed keyboard/mouse delivery — no focus stealing.

```sh
/bin/bash -c "$(curl -fsSL https://cua.ai/driver/install.sh)"
```

Grant **Accessibility** permission when prompted.

</Step>

<Step>

### (Optional) computer-server — for the streaming LLM demo

The streaming demo (`demo_live.py`) observes the accessibility tree through
cua's computer-server:

```sh
git clone https://github.com/trycua/cua
cd cua/libs/python/computer-server
uv run python -m computer_server --port 8765
```

Grant your terminal **Accessibility** permission.

</Step>

</Steps>

## 4. Run things

```sh
# the one-liner: checks the setup, auto-starts the matcher if down, listens
uv run axstream up            # type commands ('quit' to stop)
uv run axstream up --voice    # speak them (needs: uv sync --extra voice)

# check the setup
uv run axstream --doctor

# one utterance -> one action (instant tier; JSON result on stdout)
uv run axstream "launch safari"

# instant-tier demo with seeded macros — slot variations, timings
uv run python demo_replay.py

# speak instead of type (local STT; needs: uv sync --extra voice)
uv run python demo_voice.py --hands-free

# stream a novel task through an LLM (needs computer-server + a key)
export GROQ_API_KEY=...
uv run python demo_live.py --task "open TextEdit and type hello world" \
  --uri ws://localhost:8765/ws
```

## 5. Tests

```sh
uv run pytest tests
```
