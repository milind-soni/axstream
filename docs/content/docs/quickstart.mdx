Everything below starts from nothing on a Mac. You need
[uv](https://docs.astral.sh/uv/) and [Homebrew](https://brew.sh). Total setup:
about ten minutes, most of it downloads.

## 1. Get axstream

```sh
git clone https://github.com/milind-soni/axstream
cd axstream
uv sync
```

## 2. See the core trick — 10 seconds, no keys, no setup

```sh
uv run python demo_dry.py
```

This replays a canned LLM response at realistic decode speed into the real
compiler and executor against a mock computer. Watch the timeline: actions
execute *while the model is still generating*, and the summary shows ~37%
wall-clock saved vs waiting for the full response. That's the streaming thesis
in one command.

## 3. Install the two local pieces

<Steps>

<Step>

### The matcher — download the fine-tuned model

```sh
brew install llama.cpp
mkdir -p ~/models && curl -L -o ~/models/lfm25-350m-axstream-Q4_K_M.gguf \
  "https://huggingface.co/milsoni201/lfm25-350m-axstream-matcher/resolve/main/lfm25-350m-axstream-Q4_K_M.gguf"
```

That's [`axstream-matcher`](https://huggingface.co/milsoni201/lfm25-350m-axstream-matcher)
(219MB) — a 350M model fine-tuned for exactly one job: *which known command is
this, and what are the variable words*, in ~100ms on Apple Silicon. It scores
**94% end-to-end** where the stock base model scores 47%. You don't need to
start it manually — `axstream up` does that for you.

</Step>

<Step>

### The executor: cua-driver

Delivers keys and clicks to apps **in the background** — no focus stealing,
your mouse stays yours.

```sh
/bin/bash -c "$(curl -fsSL https://cua.ai/driver/install.sh)"
```

Grant **Accessibility** permission when it asks — that's what lets it press
keys on your behalf.

</Step>

</Steps>

## 4. Start it and try things

```sh
uv run axstream up
```

`up` checks everything (starts the matcher if it's down, wakes the driver),
seeds a starter library of ~180 macOS commands on first run (`axstream seed`),
and listens. Try typing:

```
» open my downloads folder
  ⚡ instant [finder_go_downloads] done · match 75ms · no LLM

» take a screenshot
  ⚡ instant [screenshot_full] done · match 68ms · no LLM

» quit this app
  (refused — risky commands are gated until you allow them)
```

⚡ **instant** means a known command replayed with no LLM anywhere — that's
the tier that makes axstream feel different.

## 5. Teach it something new

Add a fast-LLM key (either provider) to `.env` in the repo root:

```sh
echo 'OPENROUTER_API_KEY=sk-or-...' >> .env    # or GROQ_API_KEY=gsk_...
```

Restart `axstream up`, then say something it doesn't know:

```
» create a new tab in firefox
  🐢 fast — the LLM watches the screen, does it, and learns it
» create a new tab in firefox
  ⚡ instant [new_firefox_tab] done · match 101ms · no LLM
```

That's the flywheel: **every command is slow at most once.** Learned macros
live in `~/.axstream/macros.json` — plain JSON, yours to inspect or edit.

## 6. Speak instead of type

```sh
uv sync --extra voice
uv run axstream up --voice
```

First run downloads a local speech model (~600MB) and asks for microphone
permission. Then just talk — pause about a second to commit, say "quit" to
stop. Everything stays on your machine: mic → local STT → local matcher →
executor. The only network call is the LLM tier for *novel* commands.

## Troubleshooting

```sh
uv run axstream --doctor    # checks matcher, driver, macro store — with fixes
uv run pytest tests         # the suite
```
